💡 Implementation Highlights

Redux Store Configuration

export const store = configureStore({ reducer: { auth: persistedAuthReducer, leaves: leavesReducer, users: usersReducer }, middleware: (getDefaultMiddleware) => getDefaultMiddleware({ serializableCheck: { ignoredActions: ['persist/PERSIST'] } }) });

Protected Route Pattern

function ProtectedRoute({ children }) { const { isAuthenticated } = useSelector( (state) => state.auth ); if (!isAuthenticated) { return <Navigate to="/login" replace />; } return children; }

Async Thunk for Login

export const loginUser = createAsyncThunk( 'auth/login', async (credentials, { rejectWithValue }) => { try { const response = await authAPI.login(credentials); toast.success('Login successful!'); return response.data; } catch (error) { return rejectWithValue(error.message); } } );

⭐ Pagination Component

const Pagination = ({ currentPage, totalPages, onPageChange }) => { return ( <div className="pagination"> <button onClick={() => onPageChange(currentPage - 1)}> Previous </button> <span>Page {currentPage} of {totalPages}</span> <button onClick={() => onPageChange(currentPage + 1)}> Next </button> </div> ); };

💪 Challenges & Solutions

Challenge: State Persistence

Maintaining user session across page refreshes

Solution: Implemented Redux Persist with localStorage, configured to persist auth state while keeping other slices in memory.

Challenge: Role-Based Routing

Different views for admin and employee roles

Solution: Created ProtectedRoute wrapper with role checking, conditional rendering in Sidebar, and server-side permission validation.

Challenge: Date Range Validation

Preventing invalid date selections and overlaps

Solution: Client-side validation with HTML5 date inputs, server-side business logic for overlap detection, clear error messaging.

Challenge: Type Safety

Ensuring type safety across Redux and components

Solution: Comprehensive TypeScript interfaces, typed Redux slices and thunks, strict tsconfig settings, and ESLint rules.
